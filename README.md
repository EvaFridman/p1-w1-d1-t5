# p1-w1-d1-t5 · class как сахар над прототипами

> Тип: объёмная (class = обёртка над прототипами) · Репозиторий: один отдельный

> Подключить `.js` к HTML через `<script>`, отсмотреть в браузере (или запустить в консоли).

## Задача

Те же секундомеры, что в прошлой задаче, но теперь через `class`. Цель — не просто переписать, а доказать, что под синтаксисом `class` работает ровно тот же прототипный механизм: методы в прототипе, класс — это функция, `instanceof` идёт по цепочке. Плюс — сравнить два синтаксиса на одинаковом поведении.

```
class — сахар над прототипами (не новый механизм)
┌──────────────────────────────────────────────────────┐
│ class User { constructor(name){ this.name=name } }   │
│ greet(){...} → лежит в User.prototype                │
│ typeof User === 'function' (докажи)                  │
│ u instanceof User → true (по цепочке прототипов)     │
└──────────────────────────────────────────────────────┘
тот же результат, что конструктор+prototype, но читаемо
```

## Дано

```js
// переписать Stopwatch из t4 на class, поведение идентичное
const labels = ["Работа", "Отдых", "Учёба"];
```

## Что нужно сделать (по релизам)

**Release 1 — класс.** Написать `class Stopwatch` с `constructor(label)` (поля `label`, `seconds = 0`) и методами `tick()`, `reset()`, `format()` — поведение ровно как у функции-конструктора из t4, включая чейнинг `tick().tick()`. Создать экземпляры, проверить, что результаты совпадают с t4.

**Release 2 — доказать, что это прототипы.** Показать: `typeof Stopwatch === "function"`; `Stopwatch.prototype.tick` существует, а `sw.hasOwnProperty("tick")` → `false`; два экземпляра делят один метод (`a.tick === b.tick`). Вывод: `class` не новый механизм, а тот же прототип под удобным синтаксисом (сослаться на свои же наблюдения из t4).

**Release 3 — instanceof и сравнение.** Показать `instanceof`: `sw instanceof Stopwatch` → `true`, `sw instanceof Object` → `true` (по цепочке), `[] instanceof Stopwatch` → `false`. Объяснить, что `instanceof` ищет `Stopwatch.prototype` в цепочке объекта. В `NOTES.md` — короткая таблица «функция-конструктор vs class»: что одинаково (механика), что различается (синтаксис, читаемость).

## Ожидаемый результат

- Класс ведёт себя ровно как функция-конструктор из прошлой задачи: те же тики, чейнинг и формат времени.
- Проверки подтверждают, что под `class` работают прототипы: сам класс по типу является функцией, метод лежит в прототипе (не собственное свойство экземпляра), экземпляры делят один метод.
- Проверка принадлежности показывает, что экземпляр создан этим классом (и попутно является объектом), а посторонний массив — нет.

## Критерии приёмки

* Класс воспроизводит поведение конструктора из t4 (включая чейнинг); результаты совпадают.
* Доказано, что `class` — функция и методы в прототипе (тот же механизм, что t4).
* `instanceof` продемонстрирован и объяснён через цепочку; в `NOTES.md` — сравнение двух синтаксисов.
